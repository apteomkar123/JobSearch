7. FIX — RESUMES SHOWING "RESUME LOADING" INDEFINITELY (2026-06-04)
   The render() function checked Object.keys(RESUMES).length === 0 to decide
   whether to show "Resumes loading..." vs "No resume for this role yet".
   Since the resume chunk files (resumes_0.json through resumes_11.json) don't
   exist, all fetches return 404, RESUMES stays empty {}, and the check always
   evaluated to true — showing "Resumes loading..." forever.
   Fix: Added resumesLoaded boolean flag to js/state.js. Set to true after the
   async chunk loading completes (whether chunks loaded or not). Updated render()
   and toggleRecent() in js/ui.js to use !resumesLoaded for the loading state.
